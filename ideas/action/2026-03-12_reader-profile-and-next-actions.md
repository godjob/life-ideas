## 読書傾向から見えること

### 1. 「自己最適化」への強い欲求
自己啓発59冊（35%）が最多。「習慣」「時間管理」「思考法」が頻出テーマ。ハイライント数TOP3が「時間・心・思考法」系。「もっとうまくやれるはず」という内発的動機が強い。

### 2. 知的好奇心と大局観
ハラリ（サピエンス全史、ホモデウス、NEXUS）を3冊読了。山口周も複数。「なぜ」「どこへ向かうのか」という大きな問いへの関心。技術者でありながら哲学・人文系を読む。

### 3. エンジニア思考と「Be Lazy」志向
「世界一流エンジニアの思考法」に147ハイライト（最多タイ）。システム思考・効率化・自動化への親和性。コードより「仕組み」を作ることへの関心。

### 4. 矛盾する欲求（テンション）
「多動力」（堀江貴文）を読みながら「限りある時間の使い方」も読んでいる。「自由になりたい」と「うまく管理したい」が共存。製造業システム管理という「安定・堅実」な職業にいながら、「もっと自由に・広く・速く」への憧れがある。

### 5. 「アウトプット・発信」への潜在欲求
文章術を4冊読んでいる（書く習慣、文章で飯を食う方法、言葉にできるは武器になる）。インプットだけでなく、自分の知見を外に出したい欲求がある。

## 課題：インプットがアウトプットに繋がっていない

169冊すべての「自分の考察」が空白。ハイライトを引く段階では記憶への符号化は起きているが、「自分の言葉で書き直す」「他の知識と接続する」がないと数ヶ月後には残らない。

文章術を4冊読んでいるのに書いていない。「何を書けば価値があるか」という完璧主義的なフィルターが動けない原因になっているケースが多い。

## 実行フェーズ設計（2026-03-15 更新）

> ランニングが続いている理由と同じ「仕組み」で習慣化する。
> 既存の git scrap ワークフローに乗せることで摩擦をゼロにする。

| フェーズ | 期間 | アクション | トリガー |
|--------|------|-----------|---------|
| Phase 1 | 今〜4週間 | 週1回・本の考察を書く | **日曜 git scrap 後に自動で問いが届く** |
| Phase 2 | 5〜12週間 | 社内の繰り返し作業をAIでPoC | Phase 1 の継続実績から |
| Phase 3 | 3ヶ月以降 | 発信の場を1つ決めて書く | Phase 2 の実績から |
| 番外 | 今週中に1回 | 子どもとAIを一緒に使う | 一回限りでOK |

### 仕組み（Phase 1）

日曜日に `git scrap` を実行すると、スクリプトが自動で：
1. 直近7日に読んだ本を検出（なければ最後に読んだ本）
2. 代表ハイライト3件 + 振り返りの問い2個をターミナルに表示
3. 入力を受け取って `## 自分の考察` に書き込み → コミット

**所要時間: 5〜10分。ランニング後のストレッチと同じ感覚で。**

## アクション詳細

### ✅ Action 1：読書考察（Phase 1 — 自動化済み）

**目標:** 週1冊、`## 自分の考察` を書く

- トリガー: **日曜 git scrap** → ターミナルに問いが自動表示される
- 入力: 1〜5行でいい。整形は Haiku が行う
- 保存先: `ideas/books/YYYY-MM-DD_slug.md` の考察セクションに自動書き込み

**進捗:**
- [ ] Phase 1 初回（次の日曜に git scrap を実行）
- [ ] 4週間連続（1ヶ月）
- [ ] 考察ありの本が10冊になる

---

### ✅ Action 2：AI × 本業の「先駆者ポジション」（Phase 2）

**目標:** 製造業システム管理 × AI活用の実践者として社内外で認知される

製造業のシステム管理者でAIエージェントを実用している人は希少。社内の繰り返し作業（日報・障害対応手順・設備管理など）をAIで効率化し、小さく試して社内共有する。

**進捗:**
- [ ] 社内の繰り返し作業を1つ洗い出す
- [ ] AIで自動化・効率化するPoC（小さく）
- [ ] 社内で共有・提案する

---

### ✅ Action 3：週1回・自分の言葉で書く（Phase 1 に統合）

**目標:** `ideas/daily/` に週1回の書く習慣をつける

→ Action 1（読書考察）と同じトリガーで実現できる。
考察を書いたら `ideas/daily/` に連動してアウトプットが蓄積されていく。

**進捗:**
- [ ] 1ヶ月継続

---

### ✅ Action 4：子どもへの「AI教育の最前線」（番外・今週中）

**目標:** 「AIを使いこなせる親」として子どもに実践を見せる

中学生の娘・小学生の息子に「こういう質問の仕方をすると精度が上がる」を見せるだけでいい。一回やれば十分。

**進捗:**
- [ ] 子どもと一緒に Claude/AI を使う機会を1回作る

---

### ✅ Action 5：アウトプットの場を1つ決める（Phase 3）

**目標:** portfolio を使い始める

169冊の知識 × AI活用 × 製造業システム管理という組み合わせは、書けば確実に読まれる。Phase 2 の実績ができてから決める。

**進捗:**
- [ ] 発信の場を1つ決める（note / Zenn / 社内Wiki / life-ideas）
- [ ] 最初の1記事を書く
