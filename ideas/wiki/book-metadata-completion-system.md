# 書籍メタデータ補完システム：OpenBD→NDL→Google Booksの3段階フォールバック

## 概要

書籍のメタデータ（タイトル、著者、出版社、ISBN等）を複数のデータソースから段階的に取得するシステムです。OpenBD（日本の書籍情報データベース）を第一選択肢とし、データが不足する場合はNDL Search APIにフォールバック、さらに不足する場合はGoogle Books APIを利用することで、170件中132件（77.6%）の補完率を達成しました。このアプローチにより、単一のデータソースの制限を回避し、継続的にメタデータの品質を向上させられます。

## 主要な知見

- **3段階フォールバック戦略の有効性**：OpenBD → NDL Search API → Google Booksの優先順位付けにより、日本の書籍情報は国内ソースで優先的に取得でき、国際書籍も網羅可能
- **補完率77.6%の達成**：170件中132件のメタデータ補完により、大部分のユースケースをカバー。残り38件は絶版書や非商業出版物などの希少書籍
- **import_kindle.pyの再実行バグ修正**：load_existing_meta関数の実装により、既存メタデータの消失を防止。システムの再実行を安全に行え、新規書籍のメタデータ追加と既存データの保護を両立
- **APIの特性理解**：各APIの応答形式、レート制限、カバレッジの差異を理解することで、効率的なフォールバック設計が可能
- **段階的検索による効率化**：最初のAPI呼び出しで完全なメタデータが取得できる場合が多く、不足分のみ後続APIに問い合わせることでAPI呼び出し数を最小化

## システム設計

### fill_book_meta.pyの構成

書籍メタデータ補完システムの中核はfill_book_meta.pyで、以下の機能を実装しています：

1. **既存メタデータの保護**：load_existing_meta関数により、再実行時に既存のメタデータが消失するバグを防止
2. **段階的フォールバック**：
   - 第1段階：OpenBD APIに問い合わせ（ISBN検索）
   - 第2段階：NDL Search APIに問い合わせ（タイトル・著者検索）
   - 第3段階：Google Books APIに問い合わせ（最後の手段）
3. **メタデータの正規化**：各ソースから取得したデータを共通スキーマに変換

### 各データソースの特性

**OpenBD（Open BizCode Database）**
- 日本の出版社が参加する公開書籍情報データベース
- ISBN検索による高精度な取得
- 国内出版物の網羅性が高い
- レート制限が緩い

**NDL Search API（国立国会図書館サーチ）**
- 日本の公的図書館データベース
- タイトル・著者による柔軟な検索
- 絶版書や古い書籍の情報も保持
- 複数の検索キーワード対応

**Google Books API**
- 世界規模の書籍情報カバレッジ
- 国際書籍や翻訳書に強い
- レート制限が厳しい
