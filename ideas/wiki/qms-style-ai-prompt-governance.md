# QMS様式のAIプロンプト統治：製造業の手順書運用をClaude Codeに適用

## 概要

製造業の品質マネジメントシステム（QMS）では、手順書を中央集約的に管理し、全員がそれに従うことで品質を担保する。Claude Codeの設定ファイル（CLAUDE.md）の仕組みは、このQMS思想そのものであり、AIに対する設計規約・セキュリティ要件・業務手順を一度記述すれば、以後のすべての対話で常に遵守される。これにより、製造業レベルの再現性と効率性をソフトウェア開発・業務自動化に適用することが可能になる。

## 主要な知見

- **CLAUDE.md = デジタル時代の手順書**
  - QMSでは手順書を一元管理し全員が従う。CLAUDE.mdは毎回読み込まれるため、設計規約やセキュリティ要件を一度記述すれば、以後のすべての対話でAIが自動的に遵守する仕組み
  - 人間の監督者が不要になるほどの「自動的な準拠性」が実現される

- **並列実行による計算効率化**
  - 複数のサブエージェントを並列実行しても計算コストが大幅に下がる設計により、保守業務（監査・ドキュメント・テスト・パッチ適用）を同時進行できる
  - 間欠トレーニング（複数の異なる強度を並行実施）と同じ効率性で、従来の順序立った業務プロセスを破壊できる

- **権限設定による確認操作の削減**
  - 適切な権限設定により、システム管理者による属人的なチェック作業を9割削減できる
  - 現場のボトルネック（確認待ち）をAIが自動化する具体的な道筋を示す

- **QMSの監査・改善ループの自動化**
  - 製造業で定期的に行われるQMS監査（内部監査）や改善提案システムを、AIエージェントが継続的に実行できる
  - [CLAUDE.md自動育成：Hook機械化と無限ループ対策](claude-md-auto-cultivation-hooks.md) により、手順書そのものを日次で改善する仕組みが構築される

## 本文

### QMSの原理をAI統治に応用する意味

製造業のQMS（品質マネジメントシステム）は、以下の原則で成り立つ：

1. **中央集約的な文書管理**：手順書は一箇所に統一管理され、バージョン管理される
2. **全員の準拠性**：全従業員が常に最新の手順に従うことで、ばらつきを排除する
3. **監査と改善**：定期的な内部監査を通じて、手順の有効性を検証し改善する
4. **トレーサビリティ**：誰がいつ何をしたかを記録し、問題発生時に原因を追跡できる

Claude Codeの設定駆動ワークフロー（[Claude Code設定駆動ワークフロー：CLAUDE.mdの設計規約自動遵守と保手業務の並列化](claude-code-configuration-driven-workflow.md)）は、AIに対してこれらの原則を完全に適用する仕組みである。

### CLAUDE.mdを「デジタル手順書」として機能させるための要件

CLAUDE.mdをQMSの手順書同様に機能させるには、以下の要件を満たす必要がある：

1. **常時読み込みと強制遵守**
   - 毎回の対話開始時にCLAUDE.mdが読み込まれ、その中身が対話の文脈に組み込まれる
   - AIはこの指示に対して優先度が最も高い拘束力を持つため、ユーザーからの一時的な指示より遵守される

2. **バージョン管理と変更履歴**
   - CLAUDE.mdの変更を日付付きで記録し、どのバージョンのどの指示が適用されたかを追跡可能にする
   - QMSの変更管理と同じく、意図しない変更から保護する仕組みを備える

3. **権限と責任の明確化**
   - CLAUDE.mdを変更する権限者を限定し、変更前後での動作検証を必須化する
   - システム管理者・セキュリティ担当・業務プロセスオーナーなど、役割に応じた確認ゲートを設定

4. **監査証拠の自動記録**
   - AIエージェントが実行した各種作業について、「いつ」「だれが」「何を」「どの手順に従って」実行したかを自動ログに記録する
   - これにより、問題発生時に原因を特定でき、改善活動に活用できる

### 権限設定による「自動的な確認」の仕組み

製造業では、重要な工程変更や納入判定には複数の検査者による確認が必須である。同様に、AIが実行する重要な作業についても、権限に応じた自動確認を組み込むことで、人間による手動チェックを大幅に削減できる：

- **低リスク作業**（ドキュメント更新、ログ分析など）：AIが単独で実行可能
- **中リスク作業**（テスト実行、軽微なパッチ適用）：CLAUDE.md内の事前承認ルールに従えば自動実行
- **高リスク作業**（本番環境への変更、セキュリティポリシー更新）：AIが提案を生成し、人間の最終承認を必須とする

このように権限を階層化することで、人間の監督者が日々のマイクロマネジメントに追われず、例外ケースの判断に集中できるようになる。

### 監査と改善のループ自動化

QMSでは「内部監査」により、手順書が実際に機能しているか、改善の余地がないか定期的にチェックする。これと同じく、AIエージェントに以下のタスクを定期的に実行させることができる：

1. **動作ログの定期分析**：CLAUDE.mdのルールに準拠できなかったケースや、判断に迷ったケースを自動抽出
2. **ボトルネック検出**：ユーザーからのリクエストが滞っていないか、確認待ちが発生していないか監視
3. **改善提案の生成**：現在のCLAUDE.mdで対応しきれていないシナリオを特定し、新しいルール案を提示
4. **バージョンアップの準備**：改善提案に基づいてCLAUDE.mdの新版ドラフトを作成し、人間の承認を待つ

このループが機能することで、手順書自体が「生きた資産」として継続的に進化し、組織全体の効率性と品質が向上していく。

### 並列実行による保守業務の効率化

製造業では品質維持のため、検査・監査・改善を継続的に実行する必要がある。一方、従来はこれらを順番に実行するため、全体のサイクル時間が長くなってしまう。

Claude Codeで複数のサブエージェントを並列実行することで、以下のような保守タスクを同時に進めることができる：

- **エージェントA**：ログ監査と非準拠ケースの検出
- **エージェントB**：パフォーマンスボトルネックの分析と最適化提案
- **エージェントC**：セキュリティポリシー準拠性の検証
- **エージェントD**：ユーザーフィードバックの収集と改善案の生成

これらが同時に実行されても、すべてが同じCLAUDE.mdを参照するため、矛盾が生じず、むしろ複数の視点から手順の有効性を検証できる。計算コストも大幅に削減される。

### 導入時の実装パターン

CLAUDE.md駆動の統治を導入する際の典型的なステップは以下の通り：

1. **Phase 1：基礎設定**
   - 現在の業務フロー、セキュリティ要件、設計規約を文書化し、CLAUDE.mdの初版を作成
   - バージョン管理用のリポジトリを整備

2. **Phase 2：パイロット実行**
   - 低リスクタスク（ドキュメント生成など）から開始し、AIが手順に従う動作を検証
   - ユーザーからのフィードバックを基に、CLAUDE.mdの微調整を実施

3. **Phase 3：権限段階化**
   - 段階的に高リスクタスク（テスト実行、デプロイ支援など）をAIに委譲
   - 各段階で監査ログを確認し、想定外の動作がないか検証

4. **Phase 4：監査ループの確立**
   - AIによる定期監査・改善提案の仕組みを実装
   - CLAUDE.md自体の改善サイクルを確立
