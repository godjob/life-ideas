# CLAUDE.md自動育成：Hook機械化と無限ループ対策

## 概要

CLAUDE.mdに「自動実行してください」と記述しても、Claude（AIも人間も）は指示を忘れてしまう。この課題を解決するために、Hookを使って機械的に強制実行する仕組みを構築することが正解である。SessionEnd、PreCompact、PreToolUseなどの複数のHookタイミングを活用し、無限ループ対策として環境変数フラグパターンを導入することで、堅牢な自動育成システムが実現できる。

[Claude Codeの長期記憶システム設計](claude-long-term-memory-design.md)との検証を通じて、CLAUDE.md + auto memoryの実装パターンが確立され、Hook機械化はこの長期記憶を活用した継続的改善の中核メカニズムであることが明らかになった。

## 主要な知見

- **Hookによる強制実行**：単なる指示では忘却が避けられないため、Hookメカニズムで機械的に強制実行する設計が本質的解
- **複数タイミングの活用**：SessionEnd と PreCompact の両方で分析を実行することで、圧縮前の重要情報を漏れなく拾える
- **スキル使用計測への応用**：[PreToolUseフック](hook-design-patterns-ai-workflow.md)を活用することで、自分のスキル使用頻度を継続的に追跡・改善できる
- **無限ループ対策パターン**：環境変数フラグを用いた制御メカニズムは汎用性が高く、他のHook設計にも転用可能
- **CLAUDE.mdとの相乗効果**：[CLAUDE.md統治](claude-md-governance.md)と組み合わせることで、AIへの経営判断基準が日次で自動改善される
- **長期記憶との統合**：[Claude Codeの長期記憶システム](claude-long-term-memory-design.md)と組み合わせることで、単なる自動実行ではなく、学習と改善を含む知識蓄積型の自動育成が実現できる

## 実装原理

### Hook機械化の背景

人間の指示依存では、いくら「自動実行してください」と書いても実際には実行されない。これは認知バイアスや注意資源の限界による普遍的な問題である。Hookを活用すれば、この依存を完全に排除できる。

```
指示に頼る設計 → 忘却と手動実行の繰り返し
Hook機械化 → 条件到達時に自動強制実行 → 継続的改善
```

### 複数タイミングの設計

- **SessionEnd**：セッション終了時に全体の分析を実行
- **PreCompact**：コンテキスト圧縮前に重要情報を抽出
- **PreToolUse**：ツール（スキル）実行前にメタデータを記録

このように複数のHookを組み合わせることで、情報漏れを防ぎつつ、多層的な自動育成を実現できる。詳細な設計パターンについては[Hook設計パターン：SessionEnd・PreCompact・PreToolUseの活用法](hook-design-patterns-ai-workflow.md)を参照。

### 無限ループ対策の仕組み

無限実行を防ぐために、環境変数フラグパターンを導入する：

```
HOOK_EXECUTION_FLAG=false（初期値）
  ↓
Hook トリガー時に true に設定
  ↓
実行処理中は flag=true のため、再トリガーを無視
  ↓
処理完了後に false にリセット
```

この制御フローにより、同一条件下での重複実行を防ぎながら、次セッション以降の実行を確保できる。

### CLAUDE.mdとの統合

CLAUDE.mdファイルに以下の構造を組み込むことで、Hook機械化の効果を最大化する：

```markdown
## 自動実行タスク

### SessionEnd時実行タスク
- コンテキスト分析の実行
- スキル使用統計の更新
- 改善アクションの抽出

### PreCompact実行タスク
- 圧縮前の重要データ抽出
- メモリ永続化

### PreToolUse実行タスク
- ツール実行前のメタデータ記録
- 使用頻度追跡
```

こうした明示的な記述により、Hook処理が何をすべきか常に参照可能になり、AIの自動育成がより確実に機能する。

## 設計パターンの応用例

### パターン1：スキル改善の自動ループ

```
PreToolUse Hook → スキル実行メタデータ記録
    ↓
SessionEnd Hook → 実行結果の分析・学習
    ↓
CLAUDE.md更新 → 改善指針の反映
    ↓
次セッション：改善されたスキル設定で実行
```

### パターン2：コンテキスト圧縮と情報保存の最適化

```
PreCompact Hook → 重要情報を自動抽出
    ↓
Auto Memory に永続化
    ↓
圧縮実行（情報喪失を最小化）
    ↓
新セッション開始時に長期記憶から復元
```

### パターン3：複合Hook制御

複数のHookを順序付けて実行することで、より精密な自動育成を実現：

```
PreToolUse（記録）→ SessionEnd（集計）→ PreCompact（保存）→ CLAUDE.md更新
```

この順序により、ツール実行から改善反映までの全サイクルが自動化される。

## 無限ループ対策の汎用パターン

環境変数フラグパターンは以下の場面で転用可能：

- **重複実行防止**：同一Hookトリガーによる繰り返し実行の抑制
- **レート制限**：実行頻度を制御し、リソース消費を最適化
- **段階的実行**：Hook実行の優先度付けと順序制御
- **エラーハンドリング**：失敗時の再トライ上限設定
