# lifeリポジトリの進化：GitHubからMarkdown + Claude Codeへ

## 概要

GitHubで人生を管理するというコンセプトから始まったlifeリポジトリは、単なるタスク管理ツールから、Markdown + Claude Codeを活用した統合的なライフシステムへと進化しました。このページでは、その進化の軌跡と、システム管理者視点での実装の工夫について記録します。

興味深いことに、このlifeリポジトリ自体が「非エンジニアがAIと一緒に自分仕様ツールを育てる」という、個人的な実践例となっています。

## 主要な知見

- 「[GitHubで人生を管理する](github-life-management.md)」（Zenn記事）が現在のlifeリポジトリのコンセプトの原点
- GitHubのIssueを「人生のタスク」として扱う発想は、従来のライフハック手法との大きな違い
- 「仕事と同じ操作で脳の切り替えが不要」という効果は、システム管理者にとって特に有効
- Markdown形式への移行により、テキストベースの柔軟な管理が可能に
- Claude Codeの統合で、AIエージェント的なサポート機能が追加され、運用効率が大幅向上
- [環境エンジニアリング](environment-engineering.md)の考え方が、個人のライフシステム構築にも応用可能
- チャット版Claudeで仕様・プロンプトを練り、Claude Codeで実行という使い分けは、既に実践できている正しいアプローチ

## 進化のステップ

### 第一段階：GitHubベースの運用

当初、GitHubのIssue機能を活用した理由は明確でした。仕事で日々使用しているGitHubのワークフローをそのまま人生管理に適用することで、コンテキストスイッチングを削減できます。これは[AIオーケストレーター](ai-orchestrator-role.md)としての効率化戦略と同じ思想です。

### 第二段階：Markdown + Claude Codeへの移行

GitHub Issueの制限を感じるようになり、より自由度の高いMarkdown形式への移行を決断しました。この転換により：

- ファイルベースの管理で、バージョン管理がより直感的に
- Markdownの柔軟性により、複雑な構造化情報も表現可能
- Claude Codeとの連携で、AIによる自動処理の幅が拡大

### 第三段階：AIエージェント統合

[AIエージェント運用展開](ai-agent-operations.md)の知見を応用し、lifeリポジトリにもエージェント的な機能を組み込みました。これにより、単なる記録媒体から、主動的にサポートしてくれるシステムへと進化しています。
