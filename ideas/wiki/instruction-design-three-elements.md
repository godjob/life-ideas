# 指示設計の3要素フレームワーク：背景・目的・期待アウトプット形式

## 概要

AIへの指示出しの質は、人間相手のマネジメント能力に直結する。Claude Codeなどの忖度しないAIモデルは曖昧な指示をそのまま低品質なアウトプットに変換するため、「背景・目的・期待アウトプット形式」の3点セット指示法が必須となる。この指示設計フレームワークを習慣化することで、AIとの相互作用を通じた高速フィードバックループが実現し、人間チームへのマネジメントスキルも自然に向上する。

特に重要な洞察として、**現場の専門知識とAIの役割分担**がある。税理士がAIを使うと「実務的に正しいもの」ができるのに対し、エンジニアが使うと「技術的にすごいもの」ができるのは、ドメイン専門知識の有無による。「何を自動化すべきか」「何を人間が確認すべきか」の線引きは[ドメイン専門知識とAIの境界設計](domain-expertise-ai-boundary-design.md)によってのみ決定できるのである。

さらに、[コンテキスト設計によるリーダーシップ](context-design-leadership-impression-management.md)の視点から見ると、指示の品質は「何を指示するか」と同等に「なぜそう判断したのか」という背景説明の明確さに左右される。部下や同僚に「任せる」と「丸投げ」が区別されるのは、指示に込められた文脈と判断根拠の透明性にあり、これは対人マネジメントにおいても、AIマネジメントにおいても不変の原則である。

## 主要な知見

- **忖度しないAIによる露出効果**：Claude Codeは人間のように「空気を読む」ことがないため、曖昧な指示がそのままアウトプット品質の低下に直結する。人間チーム相手では隠れていた指示の問題が、AIとの対話で即座に可視化される
- **3要素セットの必須化**：「背景（Context）・目的（Purpose）・期待アウトプット形式（Expected Output Format）」の3点が、効果的な指示の最小単位として機能する
- **高速フィードバックループ**：人間相手なら半年かかる学習サイクルを、AIなら1週間で回転させることができる。「任せすぎ→痛い目→ちょうどいい塩梅」の振り子経験が加速化される
- **副次効果としてのマネジメント向上**：AIへの指示を体系的に書く習慣が、人間チームへの引き継ぎ資料やマニュアル品質の向上につながる。同時に、指示内容と並んで「判断背景の説明」を丁寧にすることで、チーム内での信頼醸成が加速される
- **CLAUDE.mdによるナレッジ資産化**：「AIへの業務マニュアル」として[CLAUDE.md統治](claude-md-governance.md)を整備する過程が、組織全体の知識体系を明文化する契機となる
- **現場知識による境界線引き**：AIは高速で実行できるが、「何を自動化すべきか」という判断には現場の専門知識が必須。月24時間削減といった具体的な成果は、人間が設計した除外ルール（借入金・社保・給与など7種類）によって初めて実現される
- **文脈設計による信頼形成**：~~「丸投げ」と「任せる」の違いは指示内容では判別不能~~→「丸投げ」と「任せる」の違いは、指示に含まれた背景説明と判断根拠の透明性によってのみ決まる。背景要素で「なぜこの判断をしたのか」を明文化することで、受け手に安心感が生まれ、組織内の信頼資本が蓄積される

## 背景要素（Context）の設計

背景要素は、指示の前提条件となる情報を提供する。これは単なる説明ではなく、AIが判断を下すために必要な文脈の全体像を示すものである。同時に、人間の部下やチームメンバーに対しても、指示の根拠を理解させ、「信頼できるマネジメント」として認識させるための最重要要素である。

```
悪い例: 「レポートを作成してください」
良い例: 「当社は製造業で、毎月の生産実績をスプレッドシートで管理しており、
        経営陣が意思決定する際に最新の数字が必要になります。
        背景として、先月の生産効率が計画比95%だったため、
        経営層は今月の傾向を早期に把握したいという経営判断です。」
```

背景要素に含めるべき情報：

- **業務コンテキスト**：その業務が組織内でどのような位置付けにあるか
- **ステークホルダーと制約条件**：誰がこのアウトプットを使うのか、どのような制約があるか
- **現在の課題**：なぜこの指示が必要になったのか
- **判断根拠**：なぜこのタイミング、この優先度でこの業務を依頼するのか（信頼形成に不可欠）
- **既存の仕組みやデータ**：参照すべき資料や前提となるシステムは何か
- **ドメイン固有の用語や慣行**：業界・組織特有のルールや定義

具体的な背景がAIの判断精度を飛躍的に向上させる。「この報告書は経営陣向けだから簡潔に」「この処理は月末締めの翌営業日までに完了必須」といった背景情報があれば、AIが自動的に優先度やトーンを調整する。

同時に、背景を丁寧に説明する習慣は、人間チーム相手の場合、「何をするか」と「なぜそうするか」が明確に区別され、受け手の納得度と実行品質が向上する。[透明性とコミュニケーション](transparency-communication-trust-foundation.md)による信頼醸成の基盤となるのである。

## 目的要素（Purpose）の設計

目的要素は、「なぜこのアウトプットが必要か」「それによって何を実現したいか」を明確にする。目的と背景の違いは、背景が「事実」であるのに対し、目的が「望ましい状態」である点である。

```
悪い例: 「データを整理してください」
良い例: 「経営陣が3日以内に月間生産効率の前月比を判断できるようにすること。
        この判断を通じて、来月の生産計画をどの程度抑制すべきかの経営判断が
        必要になっているため、早期の数字提示が事業継続に直結しています。」
```

目的要素に含めるべき情報：

- **最終的な意思決定やアクション**：このアウトプットを使って誰が何を決めるのか
- **成功の定義**：どのような状態になれば「うまくいった」といえるか
- **時間軸**：いつまでにこれが必要か、どの時点での情報が有用か
- **優先度**：複数の要件がある場合、何が最も重要か
- **副次的な制約**：セキュリティ、監査対応、コンプライアンスといった隠れた要件
- **事業へのインパクト**：この指示がどのように組織の目標達成に貢献するのか

目的が曖昧だと、AIは技術的には完璧だが実務的には使えないアウトプットを生成する。例えば「全データを網羅的に」という指示なら見栄えのよい分厚いレポートができるが、「3分で読める意思決定資料」という目的なら全く異なるフォーマットになるべきである。

目的を明確にすることは、受け手（人間であれAIであれ）に対する配慮を示し、「あなたの時間を尊重している」というメッセージとなる。このメッセージの積み重ねが、[コンテキスト設計によるリーダーシップ](context-design-leadership-impression-management.md)における信頼形成の源となる。

## 期待アウトプット形式（Expected Output Format）の設計

期待アウトプット形式は、「どのような形でそれを渡してほしいか」を詳細に指定する。これは指示の最も具体的な部分であり、AIが実際に何をするかを決定する。

```
悪い例: 「グラフを作成してください」
良い例: 「過去12ヶ月の月間生産量を横軸を月（1月～12月）、縦軸を生産量（単位：個）
        とした折れ線グラフで、Excelの新規シートに作成。
        グラフタイトル『月間生産実績推移』、凡例は不要。
        シート名は『生産実績グラフ』とする」
```

期待アウトプット形式に含めるべき情報：

- **ファイル形式と仕様**：Excel、JSON、PDF、Markdown など具体的なフォーマット
- **構造や項目**：含めるべき列、セクション、フィールドの明確なリスト
- **視覚的な要件**：グラフの種類、テーブルのレイアウト、カラースキーム
- **量的な制限**：文字数、行数、ページ数などの上限下限
- **命名規則やルール**：ファイル名、シート名、関数の書き方など技術的な細則
- **品質チェックポイント**：数値の精度、小数点以下の桁数、エラーハンドリング

曖昧な形式指定は、AIがデフォルト挙動で判断することになり、往々にして再作業が発生する。一度「期待アウトプット形式」を具体的に指定する習慣をつけると、初回クオリティが劇的に向上し、修正ループが最小化される。

## 実践的な指示テンプレート

```
【背景】
- 組織：〇〇部門
- 業務：〇〇の月次業務プロセス
- 現在の課題：〇〇が理由で時間がかかっている
- ステークホルダー：〇〇（職種）が〇〇の目的で利用
- 判断背景：〇〇という経営判断により、この指示を下している
- 参考資料：[既存の仕組みやシステムの説明]

【目的】
- 実現したい状態：〇〇ができるようになること
- 期限：〇年〇月〇日まで
- 成功の定義：〇〇という指標で測定でき、△△を達成すること
- 事業インパクト：この実現により、〇〇の効果が期待できる

【期待アウトプット形式】
- ファイル形式：〇〇
- 構成：セクション1（内容）、セクション2（内容）、...
- 具体的な例：[参考となる既存資料や実例を示す]
- 品質基準：〇〇の項目は✓を確認、数値精度は小数点第2位まで
```

このテンプレートに従って指示を構造化することで、AIとの対話効率が飛躍的に向上する。同時に、このプロセス自体が人間チームへのマネジメント能力を高める訓練となり、指示の背景を丁寧に説明する習慣が、組織内の信頼資本の形成に直結するのである。

## 実装上の注意点

**過度な詳細化の陥穽**：全てを細かく指定すればよいというわけではない。AIに過剰な制約を与えると、逆に創意工夫の余地がなくなり、「ルール通りだが実務的には微妙」といった結果になる。背景と目的を明確にしたら、形式については「推奨値」として提示し、AIの提案に対して軌道修正するのが効果的である。

**反復改善の組み込み**：初回指示で完璧を目指さない。むしろ「これくらいの指示でどうなるか試してみる」という姿勢で、1回目のアウトプットを見て「ここを直してほしい」と修正指示を入れるほうが、最終的には高品質になる。

**ドメイン知識の役割**：この3要素フレームワークはあくまで「指示の構造」であり、それぞれの要素に何を入れるかは現場の専門知識に依存する。「何が背景として重要か」「成功の定義は何か」「形式はどこまで詳細にすべきか」の判断には、AIには代替不可能な人間の専門性が必要である。

**文脈設計による信頼形成の習慣化**：人間チーム相手の場合、背景説明の充実度が「信頼できるマネジャー」か「指示だけ出す管理者」かの評判を分ける。同じ指示でも、判断根拠を明文化して伝えるほうが、受け手の納得度と心理的安全性が大きく向上する。この習慣をAI相手で磨くことで、組織内コミュニケーション全般の品質が向上する。
