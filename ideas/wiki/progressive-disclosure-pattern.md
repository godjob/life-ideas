# Progressive Disclosure パターン：段階的情報開示による効率化

## 概要

Progressive Disclosure（段階的情報開示）は、AIエージェントやシステムに必要な情報を最適なタイミングで段階的に提供するデザインパターンです。初期段階では最小限の情報のみを伝え、関連判定後に詳細を開示することで、システムの効率性と応答速度を大幅に改善します。

## 主要な知見

- **第1段階：YAMLフロントマター** → 常にシステムプロンプトに読み込まれる。「このSkillをいつ使うか」という使用トリガーだけを記載
- **第2段階：SKILL.md本文** → 関連タスクと判断された時だけ読み込まれる。完全な実行命令とコンテキストを記載
- **第3段階：references/内のリンクファイル** → 必要に応じたナビゲーション時にのみ参照される。詳細データ、参考資料、外部リンクを配置

## 詳細解説

### 設計背景

従来のAIシステムプロンプト設計では、すべての情報を一度に読み込むため、トークン消費が多く、応答時間が増加していました。Progressive Disclosureパターンは、[Skill System](skill-system.md)の実装において、この問題を解決するために採用された重要なアーキテクチャです。

### 段階的開示の利点

1. **トークン効率化** - 初期段階で不要な情報を読み込まない
2. **応答速度向上** - 最小限の判定情報で高速に意思決定可能
3. **スケーラビリティ** - スキル数が増えても初期負荷が線形増加しない
4. **メンテナンス性** - 各段階の責務を明確に分離できる

### 実装パターン

```yaml
# 第1段階：YAMLフロントマター（常時読込）
---
name: "データ分析"
trigger: "ユーザーがデータ分析を依頼した時"
priority: 5
---

# 第2段階：本文（選別後読込）
ユーザーの依頼が分析関連と判定されたら、以下の詳細命令を実行：
1. データソースの特定
2. 分析手法の選択
3. 結果の可視化

# 第3段階：参考資料（必要時参照）
references/data-analysis/methods.md
references/data-analysis/tools.md
```

## 実装上の注意点

- YAMLフロントマターには、明確で一意な使用トリガーを記載すること
- 第2段階では「How」に重点を置き、完全な実行手順を記載
- references内のファイルは、URLやIDで参照可能な形式にすること
- 定期的にログを確認し、実際に使用されているパターンを最適化すること
- 各段階の情報量を監視し、バランスが崩れていないか継続的に評価する
- ユーザーの使用パターンに基づいて、段階別の情報配分を調整する

## ベストプラクティス

- 第1段階のトリガーは簡潔かつ明確に、複数のスキルと重複しないように設計する
- 第2段階では実行可能な具体的な手順を含め、曖昧な表現を避ける
- references内の資料は、必要に応じて自動的に抽出可能なメタデータを含める
- 各スキルの使用頻度とトークン消費量を記録し、定期的に効率性を検証する
