# AI時代のWeb制作フロー：コーディング先行による設計→実装→デザイン体制

## 概要

従来の「設計→デザイン→コーディング」というWeb制作フローが逆転する新しい時代が到来している。AIの活用により、設計後すぐにコーディングを実行し、早期モックを生成してからデザインを行うことで、手戻りを激減させ、制作効率を飛躍的に高める方法論である。これにより、非エンジニアでもアイデアがあれば実装可能な環境が整備された。

## 主要な知見

- **AIツールの役割分担が重要**: [Claude ChatとClaude Codeの役割分担：効率的なワークフロー設計](claude-chat-claude-code-workflow.md)のように、Claude（チャット）は「相談・設計・プロンプト生成」、Claude Codeは「実行・ファイル操作」という明確な役割分担が最も効率的なワークフローを実現する

- **Web制作フローの逆転**: 従来の「設計→デザイン→コーディング」から「設計→コーディング（AI）→デザイン（人間）」へシフト。AIがコーディングを担当することで、早期段階で動作するモックが生成される

- **手戻りの大幅削減**: 完成度の低いデザイン段階での指摘より、動作するモックを見てから修正する方が、全体的な変更コストが低い。ステークホルダーも「見える」「動く」ものに対してより正確なフィードバックを提供できる

- **非エンジニアでも実装可能な時代**: タレントマネジメントシステムや匿名褒めアプリなど、エンジニアリングの専門知識がない者でも、アイデアとAIの活用で実装できるという現実が既に到来している

- **プロトタイピング優先の思考**: [速さが命題：検討より先にプロトタイプを出す姿勢](speed-first-prototyping.md)と同様に、議論や設計書よりも先に動作するプロトタイプを出すことで、より実質的な判断が可能になる

## Web制作フロー：従来型 vs AI型

### 従来の制作フロー
1. **要件定義・設計** → 2. **デザイン** → 3. **コーディング** → 4. **テスト・調整**

問題点：
- デザイン段階での修正が後工程に影響
- ステークホルダーが「完成度の低いデザイン」では正確なフィードバックができない
- コーディング段階での技術的課題が後から浮上
- 全体的に時間とコストがかかる

### AI時代の新しいフロー
1. **要件定義・設計** → 2. **コーディング（AI実装）** → 3. **モックデプロイ** → 4. **デザイン（人間）** → 5. **最終調整**

メリット：
- 早期段階で**動作するシステム**が存在
- ステークホルダーが実際に触って判断できる
- デザイン修正は機能確認後なので、より適切な修正が可能
- 技術的制約が明確になった状態でデザイン開始
- 総工期が短縮される
