# ワークフローのSkill化：業務手順の標準化と共有

## 概要

日々の業務に含まれる反復的なワークフローを[Skill System](skill-system.md)として定義することで、個人の暗黙知を形式知に変え、チーム全体で共有・再利用可能な資産にする手法です。「いつもやっている手順」を明文化し、他者が同じクオリティで実行できる標準化された業務プロセスへと進化させることが本質です。

## 主要な知見

- **暗黙知の形式知化**: 個人が日常的に実行している業務ワークフローをSkill化することで、属人性を排除し、チーム全体で実行可能にできる
- **再現性と品質の確保**: 定型手順をSkill定義することで、誰が実行しても同じレベルの品質を保証できる
- **git scrapワークフロー**: lifeリポジトリ運用における`git scrap`コマンドの使用パターンをSkill化すれば、複数メンバーで統一的に運用できる
- **朝刊収集レポートSkill**: 記事収集から特定フォーマットでのまとめまでの一連の手順を標準化し、継続的な情報整理業務を効率化
- **社内ポータル更新Skill**: Desknets更新の「確認→下書き→承認依頼」という定型プロセスをSkill化することで、ポータル管理業務を一貫性を持って実行
- **段階的な展開**: まずは自分の業務で検証し、効果が確認できたら他メンバーへの共有・改善を進める段階的なアプローチが有効

## 活用シナリオ

### lifeリポジトリ運用Skill: git scrap

[GitHubで人生を管理する](github-life-management.md)ための基盤となる`git scrap`ワークフロー。日々のタスク管理や思考整理を効率化する手順をSkill化することで、チーム全体が同じ方法論で個人の活動を記録・管理できるようになります。

### 朝刊収集レポートSkill

毎朝複数のニュースソースから記事を収集し、特定のMarkdownフォーマットでまとめる業務を標準化。テンプレートと実行手順を明文化することで、誰でも一貫した品質のレポートを生成できます。

### 社内ポータル更新Skill

Desknetsなどの社内ポータルシステムへの更新業務を「確認→下書き→承認依頼」という3段階の確実なプロセスに落とし込みます。チェックリスト化することで、漏れやミスを防ぎながら効率的に実行できます。

## Skill化のプロセス

1. **業務の可視化**: 現在の業務ワークフローを詳細に記録・分析する
2. **標準化**: バラつきのある手順を統一した形に整理する
3. **SKILL.md定義**: [SKILL.md仕様](skill-md-specification.md)に基づいてSkillを設計する
4. **テスト・検証**: [Skillテスト戦略](skill-testing-strategy.md)により信頼性を確保する
5. **共有・改善**: チーム内で試用して、フィードバックをもとに継続改善する
