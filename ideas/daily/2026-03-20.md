# 2026-03-20 アイデア・気づき

## git scrap からの気づき（26/03/20）

- 製造業システムの自動化・保守コストは今後数年で劇的に低下する可能性が高く、現在の高度なエンジニアスキルより『何を選ぶか』という意思決定能力が経営層に求められる時代へシフトしている。
- 国防・軍事がフィジカルAIの最大投資先となる中で、製造業のロボット・自動化技術も加速度的に民間転用される可能性があり、サプライチェーン管理やOT（Operational Technology）領域の戦略的重要性が急速に高まる。
- エネルギー（電力）がAIインフラのボトルネックになるという指摘は、製造工場の電力計画や再生可能エネルギー導入が経営リスク管理の中核となることを示唆しており、ランニング時の体調管理と同様に『持続可能性』が競争力を左右する時代になっていく。
- Claude Codeの50 Tips記事から：**worktreeで3-5並列作業**が最大の生産性向上とのこと。自分のリポジトリでも独立したタスク（index.html改修とスクリプト修正など）を並列化できる場面が多い。`/expert-consult` のagent-teams方式はまさにこの思想に近い。
- **CLAUDE.mdの「リトマステスト」**（各行に「これがないとClaudeが間違えるか？」を問う）は重要。自分のCLAUDE.mdも膨張気味なので定期的に見直す習慣をつけたい。
- **Hooksの活用余地**: PostToolUseで自動フォーマット、Notificationでコンパクション時のコンテキスト再注入など、自分のワークフローでも取り入れられる。特にコンパクション時に「現在のタスク・変更ファイル・制約」を自動再注入する仕組みは長時間セッションの品質維持に効果的。
