# 2026-03-09 アイデア・気づき

## スマイルカーブの終焉と「コ」の字型社会から得た気づき

出典: [安宅和人ブログ](https://kaz-ataka.hatenablog.com/entry/2026/02/10/132414)

### 1. 「中間工程の消滅」は製造業システム管理にも直撃する

- 調べる・比較する・叩き台をつくる・改善するという業務はAIが代替
- システム管理者として残るのは「何を導入するか判断する」「出てきた提案のダメ出し」
- 今やっていることがそのまま当てはまる → 今の役割は実は正しい方向にある

### 2. 「判断主権」という概念

- 与信モデル・採用アルゴリズム・医療優先順位などは「価値観の埋め込み」
- 社内システムも同じ: ERPやセキュリティポリシーの自動化は誰の価値観で動くか？
- **自社で判断基準を内製できない = 外部ベンダーに判断主権を渡している**
- 社内ポータル・セキュリティ運用のAI化は「内製できる範囲を広げる」視点で検討すべき

### 3. 「責任のクリーニング構造」への警戒

- AIの提案を承認するだけの「責任端子」化に自覚的になる
- 承認ボタンを押す前に「この判断の根拠を自分の言葉で説明できるか」を問う習慣

### 4. 子どもの教育への応用

- 中学生の娘・小学生の息子が社会に出る頃、中間工程的スキル（暗記・計算・整理）はAIが担う
- 育てるべきは「何がいい・悪い」「どこへ向かいたいか」という意志と価値観
- 答えを出す速さより、問いを立てる力を伸ばす

### 5. システム管理者として「コ」の字型のどこにいるか

- ディレクション側: ネットワーク設計方針・セキュリティポリシーの決定
- ダメ出し側: ベンダー提案のレビュー・障害対応の最終判断
- 中間（薄くなる側）: 定型的な設定変更・ログ確認・マニュアル作業 → 自動化推進が正解
