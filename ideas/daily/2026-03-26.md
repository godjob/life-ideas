# 2026-03-26

## Claude Code Auto Mode の評価

新しいパーミッションモード「auto-mode」をlifeリポジトリへの適用観点で評価した。

### 気づき・学び

- **パーミッションモデルの3段階**: デフォルト確認（安全だが手間）→ auto-mode（分類器が判断）→ --dangerously-skip-permissions（全許可）という連続体。lifeリポジトリはdenyリスト＋Gitフックで独自の安全設計をしているため、この連続体とは独立した軸がある
- **分類器の限界は正直に書かれている**: 「user intentが曖昧な場合は危険な操作を許可する可能性がある」「良性操作も誤ブロックされる可能性がある」と公式が認めているのは誠実だが、研究プレビュー段階での本番適用は慎重になるべき
- **allowリストの肥大化問題**: lifeリポジトリのsettings.local.jsonのallowリストが100項目超まで膨らんでいる。これはauto-modeがあれば防げたかもしれない問題。将来的にはallowリストをコア操作のみに絞り、残りをauto-modeに任せる運用が考えられる
- **Gitフックの独立性**: denyリストもauto-modeも「Claude Codeレイヤー」の話。Gitフック（pre-commit・pre-push）はGitレイヤーで動作するため、--dangerously-skip-permissionsを含めどんなモードでも影響を受けない。安全設計の最後の砦はGitフックという理解が改めて確認できた
- **日常運用への恩恵は小さい**: git scrap・books・ideas管理といった定型操作は既にallowリスト済み。auto-modeが輝くのは「新しいことを試すとき」だが、lifeリポジトリは固定パターンが多い

### AI OSとの接続

昨日考えた「AI OS」の考察との接続がある。auto-modeの分類器は「AI向けインターフェース」の一形態 — ツールの呼び出しを人間が逐一承認するのではなく、システムが意図と安全性を判断する。ただし現時点の実装はまだ粗く、「AI向けのCapabilityレイヤー」としては発展途上。
