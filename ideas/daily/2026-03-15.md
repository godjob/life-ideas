# 2026-03-15 アイデア・気づき

## AIを「社長」として雇う経営視点

- AIに作業を「指示」するのではなく、CLAUDE.mdに「経営方針」を書いて判断を「委任」する。
- 止まるべきポイント（お金・削除など）だけ明確にし、それ以外はAIに全速力で走らせる。
- 失敗したルールはその日のうちにCLAUDE.mdに書き足し、組織の「資産」に変える。
- AIとの関係は「育てる」から「信頼のプロトコルが洗練されていく」段階へ。

## AI取締役会による意思決定

- 重要な判断を1つのAIに任せず、複数のモデル（Claude/OpenAI/Gemini）に意見を戦わせ、人間が最終判断する。
- 役割分担（実装のCodex、戦略のGemini、実行のClaude）を「スキル」として定義し、定型化する。
- この「AI取締役会」の意見がわずか数分で揃う速度感こそが、AI時代の企業の競争力になる。

## 声と電話によるAIインターフェース

- 文字だけでなく「声（電話）」をインターフェースに組み込むことで、より直感的なチェックインが可能になる。
- 通話後のタスク整理に `claude -p` を使い、プロジェクトのコンテキストを維持したまま処理させる工夫。
- 日常の開発文脈（CLAUDE.md）と生活（電話・LINE）が地続きになることで、AIが真のパートナーになる。

## AIへの指示出しはマネジメントの筋トレ（逆流効果）

- Claude Codeは忖度しないため、曖昧な指示がそのままアウトプットの質の低さに直結する。人間相手は「空気を読んでくれる」だけで、指示の問題は隠れている。
- 「背景・目的・期待アウトプット形式」の3点セットが指示に必須になると、人間のチームへの指示も同じ形式になり自然にマネジメントスキルが上がる。
- AI活用で「任せすぎ→痛い目→ちょうどいい塩梅」の振り子を高速で経験できる。人間相手だと半年かかる学びがAIなら1週間で回せる。
- CLAUDE.mdは「AIへの業務マニュアル」。これを書く習慣が人間チームへの引き継ぎ資料の質を上げる副作用がある。

## 現場の専門知識×Claude Code = 実務的に正しいものができる

- 税理士がAIを使うと「実務的に正しいもの」ができる（エンジニアが使うと「技術的にすごいもの」）。
- 「何を自動化すべきか」「何を人間が確認すべきか」の線引きは現場の知識でしか決められない。
- 自動仕訳の除外ルール7種類（借入金・社保・給与等）はまさにこの線引き。AIは実行、人間は境界を設計する。
- 月24時間削減を業務ログで可視化することで、AI投資の価値が数字で見える化される。自分のlife OSにも応用できる。
