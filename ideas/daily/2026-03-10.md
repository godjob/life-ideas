## Code Review for Claude Code から得た気づき

出典: [Anthropic Blog](https://claude.com/blog/code-review)

### 1. 「速さではなく深さ」がレビューの新しい価値軸

- コード出力量が200%増でレビューがボトルネック化 → 軽いスキムが常態化
- マルチエージェントで並列探索・検証・ランク付けすることで深さを担保
- **システム管理業務への示唆**: PowerShellスクリプトやPythonツールの変更前レビューに活用できる

### 2. 「16% → 54%」という実証データの重み

- 社内導入前後の数字が明確 → AI機能の効果測定として参考になる指標設計
- 社内ツール導入提案時に「導入前後の比較数値」を揃える習慣の重要性

### 3. コスト設計（$15〜25/件）から学ぶ価値判断

- 安価な既存GitHub Actionと使い分ける設計 → 「軽量・無料」と「深い・有料」の両立
- 自社システムの自動化でも「毎回やるか・重要時だけやるか」の使い分け設計が鍵

### 4. 「承認は人間が行う」という原則の堅持

- AIはレビューして指摘するが、マージ承認は人間 → 判断主権を手放さない設計
- スマイルカーブの終焉記事と同じ構造：AIがダメ出しし、人間が最終判断する「コ」の字型

## 独立系コイルセンターの将来戦略（専門家パネル討議より）

**元ネタ**: 「スマイルカーブの終焉と『コ』の字型社会の到来」（安宅和人）

### 核心的な気づき

- コイルセンターは典型的な「中間層」→ AI時代に最も圧縮されるリスクを抱えている
- ただし**物理的な金属加工はサイバー空間で完結しない**（「閉じない世界」の砦）
- 生き残りの鍵は「加工知識の内製AI化」= 判断OSを自社に持つこと
- 外部プラットフォームに加工ノウハウを渡した瞬間、残るのはフィジカル資産だけになる

### 戦略仮説（専門家3名合意）

1. **自律工場化**: 完全自律は不要。「定常工程は無人、例外は人間」がリアルなゴール
   - ファナック山梨工場（夜間無人でロボット生産）がモデルケース
   - 段取り・保全・初回ロット立ち上げは引き続き人間が担う
2. **宇宙×防衛サプライチェーン参入**: 自律工場化 → AS9100認証 → Tier2参入のシーケンス
   - 量は少ないが「宇宙品質で作れる会社」ブランドが自動車向けの単価交渉力を上げる
3. **自動車は超ハイテン精密スリットにニッチ特化**して縮小ではなく高付加価値化

### 10年ロードマップ

```
Phase 1（1〜3年）: 加工知識のデジタル化・DX
Phase 2（3〜5年）: 自律工場化（MES・品質AI・スケジューリング統合）
Phase 3（5〜10年）: AS9100取得 → 宇宙・防衛Tier2参入
```
