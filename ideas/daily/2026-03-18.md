## Claude Code スキルの設計知見（Thariq氏 / Anthropic）

- スキルは「markdownファイル」ではなくフォルダ（スクリプト・アセット・データ込み）として考えるのが本質
- **Gotchasセクションが最重要**: Claudeが実際に失敗した場面から作った内容が最も信号密度が高い
- スキルのdescriptionはトリガー判定に使われる。要約ではなく「いつ呼ぶか」を書く
- スキルにSQLite/JSONでメモリを持たせると自己参照型のワークフロー自動化が可能
- チームでのスキル共有はリポジトリチェックイン → プラグインマーケットプレイスへとスケールする順序が自然

## CLAUDE.mdをHookで自動育成する（AppBrew）

- CLAUDE.mdに「自動実行してください」と書いてもClaude（AIも人間も）は忘れる → Hookで機械的に強制するが正解
- SessionEnd + PreCompactの両タイミングで分析することで「圧縮前の重要情報」も漏れなく拾える
- HookはPreToolUseでスキル使用計測にも使える（自分のスキルの使用頻度を追跡できる）
- 無限ループ対策の環境変数フラグパターンは汎用的で、他のHook設計にも転用できる
