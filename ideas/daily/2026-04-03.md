# 2026-04-03 アイデア・気づき

## AWSでの運用コスト（2026年現在の目安）

### ① Amazon EC2（インフラを自前管理する場合）

検証用途・MCPツールサーバーの常駐・カスタム推論サーバーに適しています。

| 用途 | インスタンス | GPU / VRAM | 料金目安 | 推奨モデル |
| :--- | :--- | :--- | :--- | :--- |
| エントリー | G4dn.xlarge | T4 / **16GB** | 約$0.53/時（月$380） | E2B / E4B |
| ミドル | G5.xlarge | A10G / **24GB** | 約$1.01/時（月$730） | 26B A4B / 31B |
| ハイエンド | G6e.xlarge | L40S / **48GB** | 約$1.86/時（月$1,340） | 31B 長文・高精度 |

> **⚠️ 誤解しやすい点**: G4dn.xlarge の T4 は VRAM 16GB です。31B Q4_0 (17.4GB) は**入りません**。31B には G5.xlarge 以上を使用してください。

**コスト削減策**:
- スポットインスタンス: 60〜90% OFF（中断を許容できる検証・バッチ処理向け）
- Savings Plans / Reserved Instance (1年): 約40% OFF（本番常時稼働向け）

### ② Amazon Bedrock（使った分だけ払う場合）

インフラ管理が不要。小〜中規模利用に最もコスト効率が良い選択肢です。

| 項目 | 料金 |
| :--- | :--- |
| 入力トークン | 約$0.25 / 100万トークン |
| 出力トークン | 約$0.40 / 100万トークン |

> **⚠️ 注意**: 上記はGemma 4 31Bクラスの**推定単価**です（2026年4月時点でBedrockへの正式提供状況は要確認）。軽量モデル（E2B/E4B）はこの1/5程度になります。

### ③ Amazon SageMaker（プロダクション向け）

オートスケーリング・マネージドなモニタリングが必要な本番環境向け。

| インスタンス | GPU / VRAM | 料金目安 |
| :--- | :--- | :--- |
| ml.g5.xlarge | A10G / 24GB | 約$1.21/時 |

EC2より若干高いですが、エンドポイント管理・オートスケールが標準で使えます。

## git scrap からの気づき（26/04/03）

- 製造業のシステム管理者視点では、Gemma 4の多言語対応（140言語）と産業用途への制限確認が重要。競合AI構築禁止条項により、内製LLMの導入可否判断に法務確認が必須となる。
- ランニング用センサーログ等の機械学習活用を検討する場合、E2B/E4Bのエッジデバイスへのデプロイ（3.2～5GB VRAM）が可能だが、Gemma Terms of Useの商用利用規制が適用されるため事前確認が必要。
- 31Bモデルの17.4GB VRAM要件はRTX 4070 Ti Super（16GB）では不足し、RTX 4090（24GB）が最低ラインという仕様は投資判断時の重要な制約。性能トレードオフとして26B MoE版（15.6GB）の検討も有効。
